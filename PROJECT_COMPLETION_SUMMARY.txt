ARCHITECTURE OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

  Batch Cycle (10-page chunks):
  
    1. Read Checkpoint         → Get last_completed_page from DB
    2. Generate Batch URLs     → Create 10-page URL batch
    3. Trigger Batch Run       → Start ParseHub with first URL
    4. Poll for Completion     → Block until done
    5. Fetch Results           → Get data from ParseHub
    6. Store with Source Page  → Tag each item with page origin
    7. Update Checkpoint       → Update progress in DB
    8. Repeat                  → Next batch starts at (max_page + 1)

  Key Features:
    • Single project throughout
    • Deterministic & traceable
    • Idempotent (safe to retry)
    • Perfect resume from checkpoint
